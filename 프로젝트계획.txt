프로젝트 계획

1. 요구사항 정의
1-1. 사용자가 대출 정보를 입력, 조회, 수정, 삭제할 수 있는 CRUD 기능 구현
1-2. 그리드 형태의 UI로 대출 정보를 한눈에 볼 수 있도록 설계
1-3. PWA 지원으로 모바일 친화성과 홈 화면에 추가 기능 제공
1-4.심플하고 단순한 디자인

2. 기술 스택 선택
2-1. 프런트엔드: React (모바일 친화적이고 PWA를 쉽게 지원함)
2-2. 백엔드: Node.js + Express (RESTful API 제공)
2-3. 데이터베이스: MongoDB (JSON 형태의 대출 정보를 다루기 적합함)
2-4. PWA: Service Worker 및 Manifest 설정 필요

3. UI/UX 디자인
3-1. Figma 혹은 Adobe XD 같은 툴을 사용해 UI/UX 디자인 목업 제작
3-2. 단순한 컬러 스킴과 직관적인 사용자 인터페이스 설계

4.개발 단계
4-1. 초기 설정: 프로젝트 구조 설정 및 기본적인 레이아웃 구현
4-2. 데이터 모델링: 대출 정보를 저장할 데이터 모델 설계
4-3. API 개발: 클라이언트에서 사용할 API 엔드포인트 구현
4-4. 클라이언트 개발: CRUD 기능 및 그리드 UI 구현
4-5. PWA 설정: 서비스 워커 및 매니페스트 파일 설정

5. 테스트
5-1. 단위 테스트 및 통합 테스트를 통해 기능 점검
5-2. 실제 모바일 디바이스에서의 사용성 테스트

6.배포
6-1. 호스팅: Vercel 혹은 Netlify를 통한 프런트엔드 배포
6-2. 서버: Heroku 혹은 AWS를 통한 백엔드 배포
6-3. 도메인 설정 및 PWA 기능 확인
